## 一、开场介绍（1 分钟）

### 旁白

> "大家好，今天为大家演示的是我们开发的操作系统智能代理——OS Agent。
> 
> 这是一个支持自然语言驱动的服务器管理工具。用户只需用日常语言描述需求，系统就能自动理解意图、执行操作并反馈结果，真正实现'去命令行化'的系统管理体验。
> 
> **系统既可直接部署于目标服务器进行本地管理，也可通过 SSH 远程协议对多台服务器进行跨环境操作，对用户而言部署位置完全透明。**
> 
> 系统支持 CLI、Web 界面、API 接口三种交互方式，具备磁盘、文件、进程、用户、系统五大管理能力，同时内置了完善的安全防护机制。
> 
> 下面让我们通过实际演示来展示系统的核心能力。"

### 演示 1：功能概览 `对应要求: 2.1.1`

**输入**:
```
你能做什么
```

**预期效果**: 系统介绍自身能力（磁盘、文件、进程、用户、系统管理等）

### 3.1 高危命令拦截 `对应要求: 2.2.1 高风险或敏感操作识别`

#### 演示 6：拦截高危命令

**输入**:
```

```

**预期效果**: 
- `rm -rf /`：LLM 识别为极高风险操作，弹出澄清确认
- `chmod 777 /etc/passwd`：被直接拦截，返回"操作已被拦截 (风险等级: CRITICAL)"

---

### 3.2 操作范围限制 `对应要求: 2.2.2 操作范围的限制与截断`

#### 演示 7：批量删除拦截

```

```

**预期效果**: 系统识别为批量操作，弹出二次确认或拦截提示

## 五、用户与远程管理完整场景（4 分钟）

> 旁白："最后，我们展示一个完整的用户管理和远程服务器操作场景——创建用户、通过 SSH 切换到远程服务器、查看远程系统信息、最后删除用户，形成完整的操作闭环。这个场景同时覆盖了用户管理（2.1.2）、安全确认（2.2.2）和 SSH 远程管理（背景说明）三项能力。"

---

### 5.1 用户创建与SSH远程管理完整流程 `前后关联：创建用户 → SSH切换 → 查看系统 → 删除用户，形成完整闭环`

#### 演示 10：完整场景（4 步操作）

**输入（第 1 步）创建用户**:
```
创建一个名叫 demo_user 的用户
```

**预期效果**: 弹出高风险确认提示，显示风险等级和原因

**操作**: 点击"确认"按钮

**预期效果**: 用户创建成功，返回用户名和随机初始密码

---

**操作（第 2 步）SSH 切换**:

在左侧面板点击"添加服务器"，填写服务器信息（使用刚创建的 demo_user 账号或已有账号）并连接

**预期效果**:
- 服务器添加成功并显示在列表中
- 连接成功，显示"已连接"状态
- 系统自动切换到远程服务器上下文

---

**输入（第 3 步）查看远程系统信息**:
```
查看系统信息
```

**预期效果**: 返回远程服务器的系统信息（而非本地信息），验证 SSH 远程管理功能正常工作

---

**输入（第 4 步）删除用户**:
```
删除用户 demo_user
```

**预期效果**: 弹出高风险确认，确认后删除用户

**操作**: 点击"确认"按钮

**预期效果**: 用户删除成功，完成整个操作闭环

---

## 前后关联设计说明

本最终版演示文案采用以下前后关联设计：

1. **文件管理闭环**（演示 4）：创建文件 → 删除文件，形成完整的操作链
2. **安全防护演示**（演示 6）：rm -rf / 触发澄清确认 + chmod 777 /etc/passwd 被 CRITICAL 拦截，展示多层次安全防护
3. **用户与远程管理完整闭环**（演示 10）：创建用户 → SSH 切换 → 查看远程系统 → 删除用户，展示完整的用户生命周期管理和远程操作能力
4. **自然语言变体**（演示 3）：使用"硬盘还有多少空间"而非标准表述"查看磁盘使用情况"，体现意图解析能力
