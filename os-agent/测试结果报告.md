## 修复记录

本次测试中发现并修复了以下问题：

1. **api.py 缺少 DATA_DIR 定义**: `export_learning_markdown` 端点引用了未定义的 `DATA_DIR`，已添加 `from pathlib import Path` 和 `DATA_DIR` 定义。

2. **test_capabilities.py 环境参数为 None**: 能力模块的 `execute` 方法需要 `env` 参数（`Environment` 对象），原测试传了 `None`，已改为使用 `EnvironmentProbe().detect()` 获取真实环境。

3. **test_intent.py 无 LLM mock**: 意图解析测试未 mock LLM，在 API Key 未配置时全部走启发式兜底导致大量失败。已重写为使用 `unittest.mock.MagicMock` 模拟 LLM 响应。

4. **test_agent_core.py 语音配置断言过期**: 语音引擎配置值已更新为 `vosk`，原断言只允许 `local/api`，已扩展允许值。

5. **test_guardian.py 确认规则格式变更**: `requires_confirmation` 现在使用 `capability_name.action` 前缀格式（如 `file.delete`），原测试只传了纯 action 名，已修正为带能力名前缀。

6. **test_graph.py 新增测试**: 新增 12 个测试用例，覆盖状态图路由、风险评估缓存、用户友好错误提示等新功能。

7. **test_learning.py 新增测试**: 新增 12 个测试用例，覆盖学习记忆的 CRUD、统计、导出、自动同步等功能。

8. **test_api.py 新增测试**: 新增 10 个测试用例，覆盖学习记忆、sudo、健康检查、聊天等 API 端点。
