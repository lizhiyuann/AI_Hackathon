## 一、基础能力 - 磁盘管理 `对应要求: 2.1.2 磁盘使用情况监测`

### TC-DISK-01: 查看磁盘使用情况
- **输入**: `查看磁盘使用情况`
- **预期**: 以 Markdown 表格展示各分区的文件系统、容量、已用、可用、使用率、挂载点
- **验证**: 表格中包含根分区 `/` 的信息，数据与 `df -h` 一致

### TC-DISK-02: 检查 inode 使用情况
- **输入**: `查看 inode 使用情况`
- **预期**: 以表格展示各文件系统的 inode 总数、已用、可用、使用率
- **验证**: 输出格式正确，数据与 `df -i` 一致

### TC-DISK-03: 查看磁盘 IO 状态
- **输入**: `查看磁盘 IO 状态`
- **预期**: 显示磁盘 I/O 统计信息（iostat 或 /proc/diskstats）
- **验证**: 输出包含磁盘设备的读写速率等信息

### TC-DISK-04: 查看挂载点
- **输入**: `查看磁盘挂载点信息`
- **预期**: 显示所有 `/dev/` 开头的挂载点信息
- **验证**: 输出包含挂载的设备名、挂载点、文件系统类型

### TC-DISK-05: 磁盘使用的自然语言变体
- **输入**: `磁盘空间还剩多少` / `硬盘快满了吗` / `我的磁盘使用率是多少`
- **预期**: 均能正确识别为磁盘使用查询并返回结果

## 二、基础能力 - 文件管理 `对应要求: 2.1.2 文件或目录检索`

### TC-FILE-01: 列出目录内容
- **输入**: `列出 /home 目录的内容`
- **预期**: 以 Markdown 表格展示目录下的文件列表（权限、大小、日期、文件名四列）
- **验证**: 表格格式正确，内容与 `ls -la /home` 一致

### TC-FILE-02: 列出当前目录
- **输入**: `查看当前目录有什么文件`
- **预期**: 显示用户 home 目录的内容
- **验证**: 路径解析为 `~` 或 `/home/<user>`

### TC-FILE-03: 搜索文件
- **输入**: `搜索名叫 config 的文件`
- **预期**: 在 home 目录下递归搜索包含 "config" 的文件名
- **验证**: 搜索结果不超过 50 条（搜索上限限制）

### TC-FILE-04: 搜索指定路径文件
- **输入**: `在 /etc 目录下搜索 hosts 文件`
- **预期**: 在 `/etc` 下搜索名为 `hosts` 的文件，以 Markdown 列表格式展示结果
- **验证**: 结果以 `- \`路径\`` 列表格式展示，包含 `/etc/hosts`

### TC-FILE-05: 查看文件内容
- **输入**: `查看 /etc/hostname 文件的内容`
- **预期**: 显示文件内容（默认前 50 行）
- **验证**: 内容与 `cat /etc/hostname` 一致

### TC-FILE-05a: 查看目录（自动识别为目录）
- **输入**: `查看 /tmp 目录`
- **预期**: 自动检测到 `/tmp` 是目录，用 `ls -la` 列出内容，以 Markdown 表格展示
- **验证**: 不报 "Is a directory" 错误，表格包含权限、大小、日期、文件名列

### TC-FILE-05b: 查看不存在的路径
- **输入**: `查看 /nonexistent/path 的内容`
- **预期**: 返回 "路径不存在" 错误信息
- **验证**: 不报 "Is a directory" 或其他无关错误

### TC-FILE-05c: 通过路径末段推断搜索文件名
- **输入**: `查找 /etc/hosts`
- **预期**: 从路径末段推断文件名为 `hosts`，在 `/etc` 目录搜索
- **验证**: 搜索结果包含 `/etc/hosts`

### TC-FILE-06: 创建目录
- **输入**: `创建一个名叫 test_dir 的文件夹`
- **预期**: 在 home 目录下创建 `test_dir` 目录，返回"目录已创建"
- **验证**: 验证信息显示"验证通过: 目标资源已确认存在"

### TC-FILE-07: 创建嵌套目录
- **输入**: `创建目录 /tmp/a/b/c`
- **预期**: 使用 `mkdir -p` 递归创建，返回成功
- **验证**: `/tmp/a/b/c` 目录实际存在

### TC-FILE-08: 创建文件
- **输入**: `件在 /tmp 目录下创建一个 test.txt 文`
- **预期**: 创建空文件，返回"文件已创建"
- **验证**: 验证信息显示"验证通过: 目标资源已确认存在"

### TC-FILE-09: 创建带内容的文件
- **输入**: `创建文件 /tmp/hello.txt 内容是 "hello world"`
- **预期**: 创建文件并写入内容
- **验证**: `cat /tmp/hello.txt` 输出 "hello world"

### TC-FILE-10: 删除文件
- **输入**: `删除 /tmp/test.txt`
- **预期**: 弹出二次确认（因为删除操作需要确认），确认后删除文件
- **验证**: 验证信息显示"验证通过: 目标资源已确认成功删除"

### TC-FILE-11: 复制文件
- **输入**: `把 /tmp/hello.txt 复制到 /tmp/hello_backup.txt`
- **预期**: 复制成功，返回"已复制: ..."
- **验证**: 两个文件内容一致

### TC-FILE-12: 移动文件
- **输入**: `把 /tmp/hello_backup.txt 移动到 /tmp/hello_moved.txt`
- **预期**: 移动成功，返回"已移动: ..."
- **验证**: 源文件不存在，目标文件存在

### TC-FILE-13: 路径自动补全
- **输入**: `列出桌面上的文件`（输入为"桌面上"）
- **预期**: 自动解析为 `~/Desktop` 路径
- **验证**: 如果 Desktop 不存在则返回相应提示

### TC-FILE-14: 相对路径处理
- **输入**: `列出 ./project 目录`
- **预期**: 自动解析为 `~/project`
- **验证**: 路径正确

### TC-PROC-02: 搜索进程
- **输入**: `查找 nginx 进程`
- **预期**: 显示包含 "nginx" 关键词的进程信息
- **验证**: 结果仅包含 nginx 相关进程

### TC-PROC-07: 检查不存在的服务
- **输入**: `查看 nonexistent 服务状态`
- **预期**: 显示服务未找到的信息
- **验证**: 不崩溃，友好提示
